## 一、题目理解与解读

### 1.1 题目背景

本题以杭州爱慷数食公司旗下自助量贩餐厅为研究对象。该餐厅采用"自选称重计价"模式，消费者按需取菜、按重量结算。目前面临两大核心矛盾：

1. **后端备货盲目**：门店缺乏科学的菜量预测方法，凭经验备货导致食材浪费或备菜不足
2. **前端供给固化**：传统固定套餐模式无法满足消费者个性化、碎片化需求

### 1.2 数据概览

题目提供三个附件数据文件：

| 附件 | 文件名 | 大小 | 内容 | 行数 |
|------|--------|------|------|------|
| 附件1 | 附件1餐厅销售流水信息表.xlsx | ~12.8MB | 订单级流水数据，含消费时间、金额、营养汇总 | 65,534行 |
| 附件2 | 附件2部分消费订单菜品具体信息表.xlsx | ~5.5MB | 菜品级明细数据，含菜品名称、价格、营养信息 | 65,535行 |
| 附件3 | 附件3数据说明.xlsx | ~12KB | 字段说明文档 | 17字段说明 |

**关键数据特征**：
- 时间跨度：2022年9月2日至2023年11月28日（约14个月，236个营业日）
- 附件1与附件2通过`indent_id`关联，但附件2仅覆盖约18.3%的订单（11,828/64,587）
- 餐厅以午餐为主（占99.2%），晚餐仅占0.8%
- 日均订单274单，日均销售额3,033元，平均客单价11.36元
- 共237种唯一菜品，平均每单含5.5个菜品
- 消费时间集中在10:00-12:00（午餐高峰）

### 2.2 问题1建模思路

**目标**：分析菜品销售分布规律与关联关系

**方法**：
1. **描述性统计**：销量排名、Pareto分析（ABC分类）、时间趋势、餐次差异
2. **关联规则挖掘**：Apriori算法，计算支持度(Support)、置信度(Confidence)、提升度(Lift)
3. **可视化**：柱状图、Pareto图、热力图、共现网络图

**关联规则指标**：
- 支持度: Support(A→B) = P(A∪B) = count(A∪B)/N
- 置信度: Confidence(A→B) = P(B|A) = count(A∪B)/count(A)
- 提升度: Lift(A→B) = P(B|A)/P(B)，>1表示正相关

### 2.3 问题2建模思路

**目标**：预测每日就餐人数、营养素需求量、销售总额

**方法**：多模型比较 + 组合预测
1. **基准模型**：历史同星期均值法
2. **SARIMA(1,1,1)(1,1,1,7)**：捕捉趋势和以7天为周期的季节性
3. **XGBoost**：利用时间特征（星期、月份）+ 滞后特征（1/3/7/14天）+ 滑动窗口统计
4. **组合预测**：按1/MAPE加权融合

**特征工程**：
- 时间特征：星期几、是否周末、月份、日期
- 滞后特征：lag_1, lag_2, lag_3, lag_7, lag_14
- 滑动统计：3/7/14日移动平均和标准差

**评估指标**：MAE, RMSE, MAPE

### 2.4 问题3建模思路

**目标**：建立利润最大化的备菜优化模型

**数学形式**（混合整数线性规划MILP）：

**决策变量**：x_{i,t,m} — 第t天第m餐次中菜品i的备菜份数

**目标函数**：
```
max Z = Σ(p_i × s_i) - Σ(c_i × x_i) - Σ(h_i × w_i) + γ × Σ(pop_i × x_i)
```

其中：
- p_i: 菜品售价
- c_i: 菜品成本
- s_i = min(x_i, d_i): 实际销售量（通过辅助变量线性化）
- h_i: 浪费成本（备菜成本 × 浪费系数）
- w_i ≥ x_i - d_i: 剩余量
- pop_i: 菜品偏好度
- γ: 偏好权重

**约束条件**：
1. 总份量约束：Σx_i ≥ D_total + S_safety（需求 + 安全库存）
2. 营养供给约束：Σ(a_ij × x_i) ≥ R_j × (1-δ)（营养素j的供给量≥需求×容忍下限）
3. 菜品多样性约束：按类别设定最少/最多菜品数
4. 单菜品上下限：l_i ≤ x_i ≤ u_i
5. 整数约束：x_i ∈ Z+

**求解器**：PuLP + CBC（开源线性规划求解器）

### 2.5 问题4建模思路

**目标**：设计10/15/20元三个价位的套餐组合

**方法**：贪心搜索 + 局部优化

**评分函数**：
```
Score = 0.30×偏好得分 + 0.30×营养均衡得分 + 0.25×利润得分 + 0.15×共购关联得分
       + 0.15×价格符合度 - 惩罚项
```

**套餐定位**：
- 10元"经济基础型"：1主食+1荤+1素
- 15元"均衡实用型"：1主食+1荤+1半荤半素+2素
- 20元"丰富营养型"：1主食+2荤+1半荤+2素+1特色

### 2.6 问题5建模思路

基于问题1-4的定量分析结果，从五个维度提出系统性策略：
1. 备菜策略（ABC分级、安全库存、预测闭环）
2. 菜品结构（生命周期管理、午/晚差异化）
3. 套餐推广（阶梯定价、动态调整、营养标识）
4. 数字化运营（数据看板、模型迭代、后厨智能）
5. 营养ESG（健康引导、浪费控制、可持续经营）

## 四、代码实现逻辑

### 4.1 项目结构

```
MathModeling/
  config.py              # 全局配置：路径、常量、营养标准、可视化参数
  data_loader.py         # 数据加载与预处理模块
  utils.py               # 通用工具函数（MAPE、特征工程、营养计算）
  problem1_analysis.py   # 问题1：ED统计 + 关联规则 + 可视化
  problem2_prediction.py # 问题2：多模型预测 + 模型比较
  problem3_optimization.py # 问题3：MILP备菜优化
  problem4_combos.py     # 问题4：贪心+局部搜索套餐设计
  problem5_strategy.py   # 问题5：五维策略分析
  main.py                # 主入口（支持 --skip/--only 参数）
  output/                # 输出目录（CSV、PNG图表）
```

### 4.2 核心模块说明

#### config.py
- 路径配置：通过文件大小特征自动匹配附件（解决中文编码问题）
- 餐次划分：午餐10:00-14:00，晚餐16:00-20:00
- 菜品分类关键词：主食/荤菜/半荤半素/素菜
- 营养参考标准：基于DRIs 2023版
- 优化模型参数：浪费成本系数(0.3)、缺货惩罚系数(0.5)、安全库存系数(0.15)

#### data_loader.py
**DataLoader类**封装了完整的数据处理流水线：
1. `_load_data()`：加载xlsx文件
2. `_clean_and_preprocess()`：日期解析、餐次划分、异常值检测(IQR)
3. `_feature_engineering()`：菜品分类、营养信息汇总、附件1+2融合
4. `_build_aggregations()`：日级汇总表 + 餐次级汇总表
5. `_build_basket()`：购物篮二值矩阵（用于Apriori）

#### problem1_analysis.py
**Problem1Analysis类**输出6张可视化图表：
- `p1_sales_distribution.png`：销量Top20 + Pareto图 + 类别占比
- `p1_temporal_patterns.png`：日趋势 + 星期模式 + 月度趋势
- `p1_meal_comparison.png`：午餐vs晚餐对比
- `p1_nutrition_analysis.png`：营养摄入分析
- `p1_association_rules.png`：关联规则散点图 + 菜品共现网络

#### problem2_prediction.py
**Problem2Prediction类**实现多模型预测框架：
- `_baseline_forecast()`：历史同星期均值
- `_sarima_forecast()`：SARIMA(1,1,1)(1,1,1,7)
- `_xgboost_forecast()`：梯度提升树
- `_ensemble_forecast()`：加权组合
- `_predict_may_2025()`：外推2025年5月工作日

#### problem3_optimization.py
**Problem3Optimization类**核心函数：
- `optimize_meal()`：单次MILP求解
  - 目标：利润最大化（收入-成本-浪费+偏好奖励）
  - 约束：总需求、营养、多样性、上下限
  - 小规模餐次（晚餐）自适应放松约束

#### problem4_combos.py
**Problem4Combos类**套餐搜索算法：
- `_greedy_search()`：200次贪心采样
- `_local_optimization()`：100次迭代局部优化（替换/添加/移除）
- `_score_combo()`：五维评分函数

#### problem5_strategy.py
**Problem5Strategy类**输出策略框架图和文本报告。

## 七、代码不足之处与可优化方向

### 7.1 当前不足

1. **菜品分类准确性**：基于关键词的分类仅有约46%的准确率（127/237归为"其他"），需要构建更完善的分类体系或使用半监督学习
2. **关联规则覆盖范围**：附件2仅覆盖18%订单，规则可能偏向有详细数据的订单群体
3. **预测模型外推能力**：2025年5月预测仅基于历史均值模式，未考虑趋势变化和外部因素
4. **晚餐数据极度稀疏**：仅11次晚餐记录，优化模型约束被迫放松，方案可靠性有限
5. **套餐搜索随机性**：贪心+局部搜索可能导致每次运行结果略有不同
6. **缺少成本数据**：菜品成本按45%率估算，可能不够精确

### 7.2 可优化方向

1. **菜品分类**：引入NLP文本分类或基于营养特征的聚类
2. **预测模型**：
   - 引入Prophet模型处理节假日效应
   - 添加外部特征（天气、节假日日历）
   - 使用LSTM/GRU进行深度学习预测
3. **备菜优化**：
   - 使用遗传算法(GA)或模拟退火(SA)处理大规模问题
   - 引入鲁棒优化处理需求不确定性
   - 晚餐使用启发式方法替代MILP
4. **套餐设计**：使用多目标进化算法(NSGA-II)同时优化多个目标
5. **在线学习**：建立增量学习机制，模型随新数据自动更新
