### 2. data_loader.py

**功能**：数据加载与预处理流水线。

**核心类**：`DataLoader`

**初始化流程**（自动执行5步）：
1. `_load_data()` — 遍历附件1(3 Sheet)、附件2(15 Sheet)，`pd.concat()` 全量拼接
2. `_clean_and_preprocess()` — 日期解析、餐次划分、异常值标记(IQR)、空列剔除
3. `_feature_engineering()` — 两轮菜品分类(关键词→营养特征)、dish_info构建、附件1↔2融合
4. `_build_aggregations()` — 日级汇总(531天)、餐次级汇总(570条)
5. `_build_basket()` — 购物篮二值矩阵(12,944×223)

**对外接口**：

| 方法 | 返回 | 说明 |
|------|------|------|
| `get_daily_data()` | DataFrame | 531天日级汇总 |
| `get_meal_data()` | DataFrame | 午餐/晚餐餐次汇总 |
| `get_dish_info()` | DataFrame | 301种菜品信息(价格/营养/成本/销量) |
| `get_basket_data()` | DataFrame | 过滤后的购物篮矩阵 |
| `get_transaction_data()` | DataFrame | 附件1+2融合明细 |

**使用方式**：
```python
from data_loader import load_all_data
loader = load_all_data()  # 一次性加载，所有模块共享
daily = loader.get_daily_data()
```

**运行时间**：约20秒（加载~18MB xlsx数据）

### 5. problem2_prediction.py

**功能**：问题二 — 多模型需求预测与2025年5月外推。

**核心类**：`Problem2Prediction`

**执行流程**（`run()` 方法）：
1. `_time_series_analysis()` — 6目标ADF检验 + ACF/PACF
2. `_build_and_evaluate_models()` — 训练4种模型(Baseline/SARIMA/XGBoost/Ensemble)
3. `_model_comparison()` — 4×6 MAPE对比柱状图
4. `_residual_diagnostics()` — Ljung-Box白噪声检验 + 残差ACF + 按星期MAPE
5. `_walk_forward_validation()` — expanding window滚动验证(step=7天)
6. `_predict_may_2025()` — **核心预测**：chinese_calendar排除假日 → SARIMA.get_forecast() → 19天预测+95%CI

**模型对比框架**：

| 模型 | 方法 | 特征/参数 |
|------|------|----------|
| Baseline | 历史同星期均值 | 无训练，直接计算 |
| SARIMA | 季节性ARIMA | order=(1,1,1), seasonal=(1,1,1,7), maxiter=100 |
| XGBoost | 梯度提升树 | ~30维特征, n=100, depth=4, lr=0.1 |
| Ensemble | 按1/MAPE加权融合 | 权重归一化，NaN模型自动置零 |

**输出**：
- `output/p2_*.png` — 6张图表
- `output/p2_may2025_predictions.csv` — 2025年5月19天预测数据

**使用方式**：
```python
from problem2_prediction import Problem2Prediction
p2 = Problem2Prediction(loader=loader)
p2.run()
```

**独立运行**：`python problem2_prediction.py`

**运行时间**：约30秒（SARIMA拟合+Walk-forward 100次XGBoost训练）

### 6. problem3_optimization.py

**功能**：问题三 — MILP午餐备菜优化。

**核心类**：`Problem3Optimization`

**MILP模型概要**：
- **决策变量**：x_i ∈ Z⁺（50种菜品整数备菜份数）
- **目标**：max Z = Σp_i·s_i - Σc_i·x_i - Σh·w_i + 0.1·Σpop_i·x_i
- **约束**：总份量(0.85D~1.30D)、营养素(±20%)、类别多样性、单品上下限(5~0.25D)
- **求解器**：PuLP + CBC, timeLimit=120s

**执行流程**（`run(prediction_csv=None)`）：
1. 加载预测数据 — 优先读取P2的CSV，回退到历史均值
2. `_select_dishes_for_optimization()` — 选择50种午餐菜品
3. 逐日调用 `optimize_meal()` — 对5个工作日分别求解MILP
4. `_plot_meal_plans()` — 备菜量/利润率/类别分布/营养雷达图

**输出**：
- `output/p3_meal_plans.png`
- `output/p3_meal_plan_detail.csv` — 50菜品×5天详细方案

**使用方式**：
```python
# 方式1：使用P2预测
p3 = Problem3Optimization(loader=loader)
p3.run(prediction_csv='output/p2_may2025_predictions.csv')

# 方式2：回退历史均值
p3.run()
```

**独立运行**：`python problem3_optimization.py`

### 7. problem4_combos.py

**功能**：问题四 — 三价位套餐优化设计。

**核心类**：`Problem4Combos`

**算法概要**：
- **第一阶段**：贪心搜索 — 按价位结构模板逐类选择(70%确定性+30%随机性)，200次独立采样
- **第二阶段**：爬山法局部优化 — 替换/添加/移除操作，100次迭代，仅接受更优解
- **评分函数**：S = 0.30·偏好 + 0.30·营养 + 0.25·利润 + 0.15·共购 + 0.15·价格符合度

**套餐结构模板**：

| 价位 | 菜品结构 |
|------|----------|
| 10元 | 主食×1 + 荤菜×1 + 素菜×1 |
| 15元 | 主食×1 + 荤菜×1 + 半荤半素×1 + 素菜×2 |
| 20元 | 主食×1 + 荤菜×2 + 半荤半素×1 + 素菜×2 + 其他×1 |

**输出**：`output/p4_combo_results.png`

**使用方式**：
```python
from problem4_combos import Problem4Combos
p4 = Problem4Combos(loader=loader)
p4.run()
```

**独立运行**：`python problem4_combos.py`

### 8. problem5_strategy.py

**功能**：问题五 — 五维度经营策略分析。

**核心类**：`Problem5Strategy`

**五维度**：
1. **备菜策略** — ABC分级(A×1.15/B×1.05/C轮换) + 预测→备菜→复盘闭环 + Z=1.65安全库存
2. **菜品结构** — 销量×单价双维评估矩阵(推广76/维持149/替换76) + 生命周期管理
3. **套餐推广** — 三层阶梯定价(15元主推50%) + 动态更新 + 营养标识
4. **数字化运营** — 数据看板 + 模型周迭代(MAPE>20%告警) + 后厨智能推送
5. **营养ESG** — 脂肪供能比优化(32.9%→<30%) + 浪费控制(319→目标159元/天)

**输出**：`output/p5_strategy_summary.png`

**使用方式**：
